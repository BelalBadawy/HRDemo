## Non-negotiable architecture rules (apply these verbatim; do not soften them)
- Domain references nothing.
- Application references Domain only.
- Infrastructure references Application and Domain.

  concrete Infrastructure types leaking upward.

  accepts and forwards a CancellationToken. No exceptions.
- Result vs. Exception boundary:
  - Expected failure states (validation errors, NotFound, Conflict, Unauthorized) are returned as
    a Result-style payload, never thrown as exceptions.
  - Exceptions are reserved for genuinely unexpected failures and are caught by a single global
    exception-handling middleware that maps them to a formatted error response.

    a specific entity) and state which side of the boundary it falls on, with the exact status
    code/result it should map to.

  Application transport contract and an API response contract.

- Mapping: manual mapping only via static extension methods, no AutoMapper/Mapster-class libraries,

- ORM migrations live strictly in Infrastructure; the API/host project must never be the migrations

## Default technology stack (use unless the user overrides)
- CQRS/mediator: a source-generator-based mediator library, not reflection-based, with generator
  package referenced ONLY in the edge/host project and abstractions-only package referenced
  wherever handlers/messages are defined.
- Validation: FluentValidation, wired as a pipeline behavior.
- ORM: EF Core + the relevant provider, ASP.NET Identity for user/role storage.
- Auth: JWT bearer, explicit `ClockSkew = TimeSpan.Zero` default.
- API surface: Minimal APIs, versioned, OpenAPI enabled.
- Health checks: liveness endpoint with no DB dependency, readiness endpoint backed by a real
  DbContext-based check (not a duplicate raw-connection-string check unless there's a stated reason
  for both).
- Testing: xUnit, FluentAssertions or equivalent, NSubstitute or equivalent, WebApplicationFactory
  for functional tests.
- Static analysis: a solution-wide analyzer package wired through Directory.Build.props, warnings
  visible but not build-breaking yet (state this is a deliberate "tighten later" roadmap item).

## Known pitfalls to avoid (apply proactively, don't wait to be asked)
- If a source-generator mediator package is used, do not reference the generator package in every
  layer — only in the edge project — and say so explicitly in the plan.
- Get pipeline behavior order right: logging/timing should wrap the whole pipeline (outermost), not
  sit inside authorization/validation, or your timing and correlation logging will be incomplete.
  Authorization should generally run before validation, not after.
- If a class library (not a web SDK project) needs `IHttpContextAccessor` or other ASP.NET Core
  abstractions, say explicitly how it gets that reference (FrameworkReference or explicit package),
  since class libraries don't get it for free.
- If Central Package Management is used, remember that per-package build metadata like
  `PrivateAssets`/`IncludeAssets` set on a `<PackageVersion>` entry in `Directory.Packages.props`
  has NO EFFECT — that metadata must be set on the actual `<PackageReference>` in the consuming
  project's `.csproj`. Call this out explicitly wherever such metadata is needed (e.g. a
  design-time-only ORM tooling package).
- If an analyzer package is listed as a version pin, make sure it is also actually wired as a real
  `PackageReference` somewhere (e.g. via Directory.Build.props) — a version pin alone does nothing.
- Don't duplicate a database health check across two different mechanisms (e.g. a raw SQL ping
  package AND a DbContext-based check) without a stated reason; pick one, prefer the one that
  exercises the actual code path the app uses.
- State where design-time tooling support (e.g. `IDesignTimeDbContextFactory`) lives, since a class
  library with no host can't otherwise construct its DbContext for CLI migration commands.
- Any place secrets could plausibly end up hardcoded (seed users, signing keys, connection strings)
  gets an explicit one-line callout — don't leave it implied.
